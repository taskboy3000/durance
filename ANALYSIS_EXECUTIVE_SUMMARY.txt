🟠 HIGH (Architectural Violations):
  3. ORM::Model is a God Object
     • 8 responsibilities (should be 1-2)
     • 22 public methods (should be 5-8)
     • Handles CRUD, queries, metadata, relationships, validation
     • Should be split into 3-4 focused classes

  4. Query Building Scattered
     • SQL construction in Model (find, all)
     • SQL construction in ResultSet (all, count)
     • SQL construction in Schema (create_table, migrate)
     • Should be unified in ORM::Query::Builder

  5. Global Registry Pattern
     • Metadata stored in package variables (%COLUMN_META, %_validations)
     • Hard to test in isolation
     • Fragile and hard to maintain
     • Should use object-based registries

🟡 MEDIUM (Code Quality):

     • _db_class_for exists in Model and Schema
     • Foreign key logic duplicated in DSL
     • WHERE clause building duplicated in ResultSet

  7. Type System Not Extensible
     • Type mapping hardcoded in ORM::Schema
     • Only supports sqlite/mysql
     • Adding new database requires code changes
